    **Step 2 — Update `closeOpenPosition` provider in trade_provider.dart to invalidate lists after close**

    The current `closeOpenPosition` provider just calls the repo. It needs to also invalidate the lists after a successful close. However, since this is a function provider (not a notifier), we need a different approach.

    Best approach: Create a new `ClosePositionNotifier` that wraps the close operation and handles invalidation:

    ```dart
    @riverpod
    class ClosePosition extends _$ClosePosition {
      @override
      FutureOr<void> build() {}

      Future<void> close({
        required String openPositionId,
        required double closePrice,
        required DateTime closeTime,
        required CloseReason reason,
      }) async {
        state = const AsyncLoading();
        state = await AsyncValue.guard(() async {
          final repo = ref.read(tradeCommandRepositoryProvider);
          final result = await repo.closePosition(
            openPositionId: openPositionId,
            closePrice: closePrice,
            closeTime: closeTime,
            reason: reason,
          );

          if (!result.isSuccess) {
            throw Exception(result.error);
          }

          // Invalidate both lists so they refresh with updated data
          ref.invalidate(openPositionsProvider);
          ref.invalidate(tradeListProvider);
        });
      }
    }
    ```

    **Keep the existing `closeOpenPosition` function provider** (other code may reference it), but the new page will use the `ClosePosition` notifier instead for automatic invalidation.
